# data/

This folder holds all raw and processed data files used by the project.

> **The two raw data files (`metadata.csv`, `train.csv`) are not tracked by Git** because of their size (>280 MB combined). Download them from the course data portal and place them here before running the notebook.

---

## Raw Data Files

### `metadata.csv` — Recipe Metadata
231,637 recipes scraped from food.com.

| Column | Type | Description |
|--------|------|-------------|
| `name` | string | Recipe name |
| `id` | string | Unique recipe ID (e.g. `R5936467`) |
| `minutes` | int | Preparation time in minutes |
| `contributor_id` | string | User who submitted the recipe |
| `submitted` | date | Submission date |
| `tags` | list(string) | Recipe tags (e.g. `["weeknight", "quick"]`) |
| `nutrition` | list(float) | `[cal, protein, total_fat, sat_fat, total_carb, sugar, sodium]` |
| `n_steps` | int | Number of preparation steps |
| `steps` | list(string) | Step-by-step instructions |
| `description` | string | Free-text description (may be null) |
| `ingredients` | list(string) | List of ingredients |
| `n_ingredients` | int | Number of ingredients |

After parsing in Section 1, the `nutrition` column is expanded into 7 named columns: `cal`, `protein`, `total_fat`, `sat_fat`, `total_carb`, `sugar`, `sodium`.

---

### `train.csv` — User–Recipe Interactions
165,226 user reviews across 62,517 recipes and 11,346 unique users.

| Column | Type | Description |
|--------|------|-------------|
| `user_id` | string | Unique user ID (e.g. `U9240752`) |
| `recipe_id` | string | Recipe being reviewed |
| `date` | date | Date of the review |
| `rating` | int | Rating from 1 to 6 (heavily skewed toward 6) |
| `review` | string | Free-text review |

**Rating distribution:** ~73% of reviews are rated 6 (the maximum). Ratings ≥ 4 are treated as "liked" interactions (~94% of all reviews).

---

## Processed Files (generated by the notebook)

These files are created when Section 3 of the notebook runs. They are small enough to optionally track in Git.

| File | Description |
|------|-------------|
| `ui_train.csv` | User–ingredient interaction table for the training split (7,935 users, ~711k rows) |
| `ui_val.csv` | User–ingredient interaction table for the validation split (1,701 users, ~151k rows) |
| `ui_test.csv` | User–ingredient interaction table for the test split (1,701 users, ~153k rows) |
| `metadata_parsed.pkl` | Parsed metadata DataFrame with nutrition columns expanded and ingredients as Python lists |

Each `ui_*.csv` has two columns:

| Column | Description |
|--------|-------------|
| `user_id` | User ID |
| `ingredient` | An ingredient from one of that user's liked recipes |

Each row represents one unique (user, ingredient) pair. Users with duplicate ingredient encounters across multiple recipes are deduplicated to one entry per unique ingredient.
